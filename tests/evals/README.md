# Eval Corpus — ADR-009 Parity Validation

Locked evaluation corpus for coherence-recovery work.  See `docs/reference-lock.md`
for pinned commits, model, and deterministic decode settings.

## Directory Structure

```
tests/evals/
├── README.md           # this file
├── prompts/            # prompt text files (one per eval case)
│   ├── sourdough.txt   # main coherence gate (22 tokens)
│   ├── short_hello.txt # fast sanity check
│   └── sliding_wrap.txt # exercises sliding-window wrap
├── reference/          # pinned reference outputs (token ids + text)
│   └── (generated by capture commands below)
└── fixtures/           # saved tensor fixtures for routine parity checks
    └── (generated by parity harness capture)
```

## Prompts

### 1. Sourdough (main coherence gate)

The canonical coherence prompt from ADR-005.  The typo ("Complrehensive") is
load-bearing — it locks the token sequence to the trajectory used for the
3094-byte baseline.

### 2. Short deterministic sanity

A minimal prompt for fast development iteration.  Should produce deterministic
output on greedy decode.

### 3. Sliding-window wrap case

A prompt long enough (or positioned to) exercise the sliding-window cache
after wrap.  Used to validate Track 2 (chronology fix).

## Generating Reference Outputs

```bash
# Sourdough reference via llama.cpp
scripts/sourdough_gate.sh "$GGUF" --min-prefix 0

# Short reference (manual)
/opt/llama.cpp/build/bin/llama-completion \
  --model "$GGUF" --file tests/evals/prompts/short_hello_rendered.txt \
  --predict 50 --temp 0 --seed 42 \
  --no-display-prompt -no-cnv -st -ngl 999 \
  > tests/evals/reference/short_hello_llama.txt
```

## Using Fixtures

Fixture-backed parity checks compare owned outputs against saved tensor
snapshots.  No model load required for routine CI.

```bash
# Capture fixtures (requires model)
hf2q parity capture --prompt tests/evals/prompts/sourdough.txt \
  --output tests/evals/fixtures/

# Check against fixtures (no model needed)
hf2q parity check --fixtures tests/evals/fixtures/
```

(Parity CLI subcommands are built during Phase 2 harness hardening.)
